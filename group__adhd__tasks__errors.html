◆ ADHD_TASKS_MAX

#define ADHD_TASKS_MAX   3

Maximum number of tasks the system can run at once.

This can be overridden by the build system, though a default is provided.

Function Documentation

◆ adhd_task_execute_next()

void adhd_task_execute_next ( TASK_PID  pid)

Execute the next instruction in a running task.

Parameters
pidThe index of the task in g_tasks to execute.

Each run of this function executes a "tick" for a given task. The kernel should call this once for each task in a loop to continue execution of user tasks.

◆ adhd_task_kill()

void adhd_task_kill ( TASK_PID  pid)

Kill the task with the given task ID.

Parameters
pidThe index of the task to kill in g_tasks.

This frees memory assigned to the task and closes its executable file. Other files may still be left open.
